**DelveAI** is a CLI-first tool (written in Rust) for managing structured **human ↔ LLM collaboration sessions as auditable trees**. Key ideas:

- **Sessions are trees**: An initial "Intent" is the root; each interaction branches into Prompts and Artifacts (context docs or accepted implementations). Accepted/rejected state controls what feeds into subsequent prompts.
- **State is filesystem-based**: Sessions are stored as directories with JSON tree files and Markdown artifacts, using short labels + random IDs for naming.
- **Three interaction modes**: non-interactive (single prompt, exit), interactive (REPL-style), and auto-interactive (LLM drives the loop, human approves).
- **Provider-backed**: Supports pluggable LLM providers (e.g. an `echo` provider for testing, `amp` for real threads).

The workspace is a Rust monorepo with 6 crates (`delve-domain`, `delve-storage`, `delve-orchestrator`, `delve-providers`, `delve-cli`, `delve-server`) plus a planned `apps/web/` frontend. It's early-stage — scaffolding, CI, and planning docs are in place.
