**DelveAI** is a **Rust CLI-first tool for managing structured Human/LLM collaboration sessions as auditable tree structures**. Key ideas:

- **Sessions are trees**: An initial intent (root node) spawns prompts, which produce artifacts (context docs or implementations). Artifacts can be accepted/rejected, and only accepted ones feed into subsequent prompts.
- **State is file-based**: Each session is stored as directories of markdown/JSON files, with labels derived from truncated text + random IDs. Code artifacts become git branches.
- **Three interaction modes**: non-interactive (single prompt→artifacts→exit), interactive (REPL-style), and auto-interactive (LLM autonomously loops: generate artifacts → review → create next prompt → repeat until done).
- **Architecture**: A Rust workspace with 6 crates (`delve-domain`, `delve-storage`, `delve-orchestrator`, `delve-providers`, `delve-cli`, `delve-server`) plus a future web frontend. It integrates with LLM providers (including Amp itself via `amp threads`).

It's essentially a **meta-orchestration layer** that wraps LLM interactions with structured session management, artifact tracking, and auditability.
