**DelveAI** is a CLI-first tool (written in Rust) for managing **human ↔ LLM collaboration as auditable session trees**. Key ideas:

- **Sessions are trees**: An initial "Intent" (root node) spawns Prompts, which produce Artifacts (context docs or implementations). Artifacts can be accepted/rejected, and only accepted ones feed future context — keeping LLM interactions efficient.
- **State is filesystem-based**: Sessions are stored as directories with JSON tree files and Markdown artifacts, identified by short labels + random IDs.
- **Three interaction modes**: non-interactive (single prompt → artifacts → exit), interactive (REPL-style), and auto-interactive (Delve loops: generate artifacts → review → suggest next prompt → repeat until done).
- **Provider-pluggable**: Supports an `echo` provider for testing and an `amp` provider that delegates to Amp threads.

The Rust workspace has 6 crates (`delve-domain`, `delve-storage`, `delve-orchestrator`, `delve-providers`, `delve-cli`, `delve-server`) plus a placeholder `apps/web/` for a future local web UI. It's early-stage — scaffolding, CI, and core CLI commands are in place.
