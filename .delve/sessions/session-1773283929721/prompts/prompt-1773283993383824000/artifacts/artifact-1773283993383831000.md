**The interactive shell is already working.** It builds, runs, and handles the full loop (browse, prompt, accept/reject, quit) with the `echo` provider. Here's what you need to use it with real LLMs:

1. **It works today** — `delve session interactive --provider echo` runs a fully functional interactive REPL loop with session tree display, prompt composition, artifact browsing, and accept/reject/complete actions.

2. **For real LLM use**, pick a provider:
   - `--provider amp` (default) — requires `amp` CLI installed and available on your `$PATH` (uses `amp threads continue --stream-json`)
   - `--provider claude` — requires `claude` CLI installed (uses `claude -p`)

3. **Sessions default to `.delve/sessions/`** in the current directory. You can override with `--sessions-dir`.

4. **To start fresh**: `delve session create --intent "your intent" --provider echo` creates a session, then `delve session interactive --session <id>` opens it interactively.

5. **Remaining unimplemented phases** (7–10 from the plan) are *not* blockers for the interactive shell — they cover git-backed artifacts, the web frontend, API server, and hardening. The interactive shell (Phase 6) is complete.
